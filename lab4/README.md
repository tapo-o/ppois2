# Отчет по лабораторной работе №4

---

### 1. Цель работы
Изучение и практическое применение архитектурного шаблона MVC (Model-View-Controller) при разработке графического интерфейса пользователя (GUI) на языке Python с использованием библиотеки Tkinter. Основная задача — обеспечить полную независимость бизнес-логики от способа её визуализации.

---

### 2. Описание архитектуры (MVC)

Проект разделен на три логических слоя:

* **Model (Модель):** Классы `Park`, `Visitor`, `Attraction`, `Ticket`. Они отвечают за хранение данных и выполнение операций (транзакции, управление очередью). Модель не содержит элементов интерфейса.
* **View & Controller (Представление и Контроллер):** Реализованы в классе `ParkGUIController` в файле `gui.py`. 
    * **View:** Визуальные компоненты Tkinter (Listbox для списков, Button для действий).
    * **Controller:** Методы обработки нажатий (например, `assign_to_attraction`), которые считывают данные из View, передают их в Model и обновляют экран.

### 4. Функциональные возможности GUI

#### 4.1. Работа со списками
Благодаря использованию флага `exportselection=False`, пользователь может одновременно выбрать посетителя в левом списке и аттракцион в правом. Это критически важно для совершения связных действий (покупка билета).

#### 4.2. Управление безопасностью
В интерфейс добавлена кнопка "Toggle Safety". Она позволяет мгновенно менять статус аттракциона. Если аттракцион имеет статус "ОПАСНО", система заблокирует попытку добавления в него посетителей.

#### 4.3. Взаимодействие с очередью
* **Кто сейчас катается?:** Кнопка вызывает диалоговое окно с именем человека, находящегося первым в очереди (на самом аттракционе).
* **Завершить поездку:** Реализует сценарий "прокатился и ушел". Человек удаляется из начала очереди, освобождая место для следующего.

---

### 5. Обработка ошибок и UX

В отличие от консольной версии, GUI-версия не требует проверки логов. Все ошибки выводятся через модальные окна `messagebox`:
* **Error:** Всплывает при попытке добавить посетителя на небезопасный аттракцион или при нехватке денег на балансе.
* **Warning:** Информирует о необходимости выбрать элементы из списков перед нажатием кнопок управления очередью.
* **Info:** Подтверждает успешную транзакцию или окончание поездки.

---

### 6. Инструкция по запуску

Программа запускается через единый файл `main.py`.

1.  Убедитесь, что установлена версия Python 3.10 или выше.
2.  Выполните команду в терминале:
    python main.py
3.  В консоли появится выбор:
    - Нажмите `1` для текстового режима (ЛР №1).
    - Нажмите `2` для графического режима (ЛР №4).

### 7. Вывод
В ходе выполнения работы была реализована полноценная GUI-система, инкапсулирующая логику модели. Архитектура MVC позволила добавить графический интерфейс без изменения исходного кода классов `Visitor` или `Ticket`, что подтверждает гибкость выбранного подхода. Были освоены приемы работы с очередями внутри GUI и механизмы обратной связи через диалоговые окна.
