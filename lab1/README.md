# Отчёт по проекту  
**Симулятор парка аттракционов**  

## 1. Предметная область

Проект моделирует работу небольшого парка развлечений (тематического парка аттракционов).  
Основные процессы:
- регистрация посетителей с кошельком;
- создание и управление аттракционами;
- продажа билетов (транзакции);
- постановка посетителей в очередь на аттракцион;
- проверка и изменение статуса безопасности аттракциона;
- управление очередью (FIFO).

Приложение полностью имитирует реальный workflow парка: от покупки билета до прохождения аттракциона, с учётом безопасности и финансов посетителя.

## 2. Основные сущности

| Сущность              | Класс                  | Основные атрибуты и поведение |
|-----------------------|------------------------|-------------------------------|
| **Парк**              | `Park`                 | Коллекции посетителей и аттракционов (словари для стабильных ID) |
| **Посетитель**        | `Visitor`              | ID, ФИО, кошелёк, метод `transaction()` |
| **Аттракцион**        | `Attraction`           | ID, название, тип, очередь, статус безопасности |
| **Билет**             | `Ticket`               | Тип аттракциона → цена (из констант) |
| **Очередь**           | `Queue`                | FIFO-список посетителей |
| **Требования безопасности** | `SafetyRequirements` | Булевый флаг `is_pass` |
| **Исключения**        | `ParkError` и наследники | `VisitorNotFoundError`, `AttractionNotFoundError`, `QueueEmptyError` |

Все сущности реализованы по принципам **ООП** (инкапсуляция, свойства, композиция).

## 3. Реализованные операции

- Создание / удаление посетителя и аттракциона
- Просмотр списков посетителей и аттракционов
- Покупка билета и постановка в очередь (с проверкой денег и безопасности)
- Переход к следующему посетителю в очереди
- Проверка, кто сейчас на аттракционе
- Изменение статуса безопасности аттракциона
- Проверка безопасности перед посадкой
- Полная обработка ошибок (неверный ID, недостаточно денег, небезопасный аттракцион, пустая очередь)

Все операции доступны через консольное меню.

## 4. Технические особенности

- **Язык и стиль**: Python 3.10+, полный PEP 8, snake_case, type hints
- **Структура данных**: словари вместо списков для стабильных ID после удаления
- **Очередь**: FIFO (`pop(0)`) + собственный класс `Queue`
- **Обработка ошибок**: кастомные исключения + `try/except` для `input()`
- **Тестирование**: 40+ тестов (pytest), покрытие ~90% (включая edge-кейсы)
- **Модульность**: каждый класс в отдельном файле + `consts.py`
- **Безопасность**: проверка `is_safe` перед посадкой
- **Расширяемость**: легко добавить новые типы аттракционов и цены через `consts.py`

## 5. Структура проекта
- `src/consts.py` — хранит цены и типы аттракционов  
- `src/Exception.py` — все пользовательские исключения  
- `src/Visitor.py` — модель посетителя (ФИО, деньги, покупка билета)  
- `src/Ticket.py` — модель билета (цена в зависимости от типа аттракциона)  
- `src/Queue.py` — реализация очереди (FIFO) для посетителей на аттракционе  
- `src/SafetyRequirements.py` — проверка безопасности аттракциона  
- `src/Attraction.py` — модель аттракциона (очередь, безопасность, тип)  
- `src/Park.py` — центральный класс, объединяет посетителей и аттракционы  
- `main.py` — консольный интерфейс и меню  
- `tests/` — тесты для проверки корректности всей логики  

## 6. Интерфейс и работоспособность

### Интерфейс
Полностью консольный. После запуска выводится меню из 13 пунктов:

1. Добавить посетителя  
2. Список посетителей  
3. Удалить посетителя  
4. Добавить аттракцион  
5. Список аттракционов  
6. Удалить аттракцион  
7. Добавить посетителя в очередь  
8. Кто сейчас на аттракционе  
9. Следующий посетитель (завершить катание)  
10. Проверить безопасность  
11. Изменить статус безопасности  
12. Показать меню  
13. Выход  

Все действия сопровождаются понятными сообщениями на русском языке.  
Ввод защищён от некорректных данных (`ValueError` при `int(input())`).

### Работоспособность

- Приложение **стабильно** работает (протестировано на Python 3.10–3.12).
- Логика соответствует предметной области: нельзя посадить на небезопасный аттракцион, нельзя оплатить билет без денег, очередь работает по FIFO и т. п.
- Удаление сущностей не ломает остальные данные.
