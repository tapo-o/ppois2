# Техническая документация проекта «Crimsonland»

Данная документация содержит описание архитектуры, функциональных модулей и игровых механик программного обеспечения, разработанного на языке Python с использованием библиотеки Pygame.

---

## 1. Архитектура программного комплекса

Проект реализован с использованием модульной структуры, что обеспечивает разделение логики обработки данных, управления состояниями и визуализации.

### Структура файлов проекта
| Файл | Назначение | Ключевые компоненты |
| :--- | :--- | :--- |
| `main.py` | Точка входа в приложение | Инициализация Pygame, запуск основного цикла |
| `src/game.py` | Ядро игры и менеджер состояний | Класс `Game`, `Button`, логика UI и меню |
| `src/entities.py` | Игровые сущности | Классы `Player`, `Enemy`, `Drop`, `Projectile` |
| `src/weapons.py` | Боевая система | Классы `Weapon`, `MeleeWeapon`, `RangedWeapon` |
| `src/utils.py` | Вспомогательные утилиты | `AssetManager`, `load_config`, `save_score` |

---

## 2. Игровые состояния (Finite State Machine)

Управление игровым процессом осуществляется через переключение состояний в классе `Game`.

| Состояние | Описание |
| :--- | :--- |
| **MENU** | Главное меню с параллакс-фоном и навигацией. |
| **PLAYING** | Активная игровая сессия (обработка физики и коллизий). |
| **RECORDS** | Просмотр таблицы лидеров (топ-10 результатов). |
| **HELP** | Экран с правилами игры и описанием управления. |
| **NEW_RECORD** | Интерфейс ввода имени игрока после завершения сессии. |

---

## 3. Управление (Input Mapping)

Логика управления реализована через опрос клавиатуры и позиционирование курсора мыши.

| Действие | Ввод | Комментарий |
| :--- | :--- | :--- |
| **Движение** | `W`, `A`, `S`, `D` | Перемещение по осям X и Y |
| **Атака** | `ЛКМ` (Mouse 1) | Использование текущего оружия |
| **Смена оружия** | `1`, `2`, `3` | Меч / Пистолет / Дробовик |
| **Навигация** | `ESC` | Возврат в меню / Пауза |

## 4. Описание ключевых классов и систем

### AssetManager (Singleton)
Реализует кэширование ресурсов для оптимизации производительности.
* **Изображения:** Автоматическая конвертация форматов (`convert_alpha`) и масштабирование.
* **Звук:** Кэширование аудиофайлов для мгновенного воспроизведения эффектов.

### Система сущностей (Entities)
Все динамические объекты наследуются от `pygame.sprite.Sprite`.
* **Player:** Обладает системой здоровья, инвентарем и логикой поворота спрайта к курсору.
* **Enemy:** Поддерживает три типа поведения (`chase`, `zigzag`, `ranged`) и анимацию смерти.
* **Drop:** Реализует механику временного существования бонусов с эффектом мерцания перед исчезновением.

### Боевая система
Построена на расчете векторов движения снарядов и круговых коллизий (`radius`) для повышения точности взаимодействия объектов.

| Параметр оружия | Описание |
| :--- | :--- |
| **Cooldown** | Задержка между атаками в миллисекундах. |
| **Damage** | Урон, передаваемый сущности при столкновении. |
| **Projectile Speed** | Скорость полета пули для дистанционного оружия. |

---

## 5. Работа с данными

Приложение использует формат **JSON** для хранения внешних данных:
1.  `config.json`: Баланс игры (характеристики врагов, шансы дропа).
2.  `scores.json`: Постоянное хранение результатов игроков в формате списка словарей.
